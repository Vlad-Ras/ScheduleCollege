# ScheduleCollege

ScheduleCollege — информационная система колледжа для составления, проверки, публикации и экспорта расписания учебных занятий.

## Соответствие ПМ.03

Проект подготовлен под требования учебной практики ПМ.03 «Сопровождение и обслуживание программного обеспечения компьютерных систем».

В проекте отражены все темы из плана практики:

1. Подбор и настройка конфигурации ПО компьютерных систем.
2. Использование методов защиты ПО компьютерных систем.
3. Проведение инсталляции ПО компьютерных систем.
4. Настройка отдельных компонентов ПО компьютерных систем.
5. Анализ рисков и характеристик качества ПО.
6. Работы на этапе поддержки ПО компьютерной системы.
7. Дифференцированный зачёт и портфолио.

Подробная матрица соответствия находится в папке `docs`:

- `ПМ03_соответствие_требованиям.txt`
- `Пояснительная_записка_ПМ03.txt`
- `PM03_матрица_соответствия.docx`

## Основные возможности

- авторизация по логину и паролю;
- роли пользователей: Admin, Dispatcher, Teacher, Student;
- справочники групп, преподавателей, дисциплин, аудиторий и временных слотов;
- создание и редактирование расписания;
- проверка конфликтов по группе, преподавателю и аудитории;
- настройка учебных дней групп;
- занятые дни преподавателей;
- импорт данных CSV/TXT/JSON;
- экспорт расписания в CSV, PDF и PNG;
- журнал действий пользователей;
- резервное копирование базы данных;
- переключение SQLite / PostgreSQL / MySQL / MariaDB;
- доступ с других ПК в локальной сети;
- сборка в один EXE для Windows;
- анализ совместимости с разными операционными системами.


## Совместимость с операционными системами

Основной готовый вариант проекта рассчитан на **Windows 10/11 x64**. Это связано с тем, что приложение собирается в один Windows EXE, использует `net8.0-windows`, Windows Forms Tray и PowerShell-скрипт публикации.

При этом интерфейс системы является веб-интерфейсом: после запуска на основном компьютере расписание можно открыть с других устройств в локальной сети через браузер. Поэтому клиентский доступ возможен с Windows, Linux, macOS, Android и iOS.

Текущая версия архива не является готовой Linux/macOS-сборкой сервера. Для полноценного серверного запуска на Linux/macOS нужно убрать Windows Forms/tray, заменить `net8.0-windows` на `net8.0` и публиковать приложение под нужный RuntimeIdentifier.

Подробно: `docs/Совместимость_с_операционными_системами.txt`.

## Запуск из Visual Studio

1. Открыть `ScheduleCollege.sln`.
2. Дождаться восстановления NuGet-пакетов.
3. Запустить проект `ScheduleCollege.Web`.
4. Открыть адрес `http://localhost:5000`.

## Запуск через командную строку

```powershell
dotnet restore
dotnet run --project .\ScheduleCollege.Web\ScheduleCollege.Web.csproj
```

После запуска открыть:

```text
http://localhost:5000
```

## Сборка одного EXE

```powershell
powershell -ExecutionPolicy Bypass -File .\build-single-exe.ps1
```

Готовый файл появится в папке:

```text
publish-single\ScheduleCollege.Web.exe
```
